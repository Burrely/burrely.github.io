Advanced Javascript

Advanced Javascript is een van de vier specialisaties van semester 2. In deze specialisatie deed ik onderzoek naar Javascript libraries, vervolgens ben ik met deze libraries aan de slag gegaan om poc’s te maken en uiteindelijk heb ik met behulp van een van deze libraries een prototype gemaakt voor mijn portfolio.

Proces

De specialisatie bestond uit drie onderdelen waarbij we elk een week kregen.

Week 1

Ik heb een longlist van libraries ontwikkelt uit libraries die mij interessant leken. Ik heb hiervoor Google en Github gebruikt, vervolgens heb ik deze lijst verwerkt in Google Sites omdat ik destijds geen portfolio site had. Vervolgens heb ik dit ingeleverd. De Google Sites pagina is hier te vinden en heeft verder buiten dit artikel geen meerwaarde.

Longlist

  1. Anime.js

    Anime.js is een library specifiek ontworpen voor animaties. Met de komst van CSS3 is het gebruik maken van Javascript voor animaties nogal vervreemd geworden voor front-end web developers. Maar wat als je meer kan doen met animaties? - Anime.js is hier het antwoord voor. Niet alleen kan je met Anime.js meer soorten animaties op een makkelijkere manier maken; je kan ook meer interactie creeëren tussen animaties en jouw front-end creatie. Denk aan bijvoorbeeld het live aanpassen van beweging, en de mogelijkheid om callbacks te hebben waardoor je iets kan laten gebeuren wanneer een animatie is afgerond, of bijvoorbeeld halverwege is!

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  2. Bideo

    Een heel simpele library, wat eigenlijk meer te vergelijken valt als een code snippet. Het laat je een video als background van een element gebruiken, bijvoorbeeld de body of dergelijke. Voornamelijk te gebruiken uit gemak.

    Om een voorbeeld te zien; klik hier.

  3. Chart.js

    Chart.js is een library waarmee je met gemak grafieken voor je data kan maken. De grafieken kunnen op vele manieren gevisualiseerd worden met Chart.js, ook geeft het je veel vrijheid in hoe je je grafieken wilt visualiseren. Deze library bespaart je veel tijd wanneer je veel bezig bent met data in je projecten.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  4. Cleave.js

    Cleave.js is een library voor gebruikersinput. Met deze library kan je detecteren wanneer bepaalde soort gegeven data iets is, bijvoorbeeld wat voor soort creditcard iemand gebruikt. Daarnaast kan deze library ook tekst formateren op een logische manier aan de hand van wat voor informatie er in een input veld tevoorschijn moet komen.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  5. Three.js

    Three.js is een javascript library die zich focused op het gebruik maken van 3D in de browser. Three.js maakt gebruik van WebGL en geeft de mogelijkheid aan developers om 3D elementen in te laden. Hiernaast heeft het een heleboel tools om interactie te bieden tussen de gebruiker en de 3D omgeving.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  6. Closure-library

    De Closure library is een library die Google gebruikt voor meerdere van hun web applicaties die nu open source is. Deze library geeft toegang tot UI Widgets, controls, methodes voor het aanpassen van elementen in je pagina, animaties etc. Vergelijkbaar met andere grotere libraries biedt het een ruim aantal opties ten koste van wat er allemaal geladen moet worden.

  7. Surveyjs

    Surveyjs is een library die surveys maken makkelijker maakt. Zelf vind ik het een beetje een vreemde library aangezien een survey maken met javascript niet echt bepaald heel moeilijk is. Maar het is en blijft een hulp tool zodat je het niet zelf hoeft te doen. De main selling points zijn dat het niet afhankelijk is van een andere service.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  8. Nanobar

    Nanobar is een lichte library die simpele, maar effectieve laad balken genereerd. Je kan deze laadbalken makkelijk beinvloeden doormiddel van het gebruik van meegegeven methods. Daarnaast kan je de looks aanpassen met gebruik van CSS. In het algemeen super handig als je een simpele laadbalk nodig hebt voor een van je projecten/designs.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  9. Javascript-state-machine

    De Javascript state machine library maakt het makkelijker om met staten van een 'object' te werken, of eerder, een tekst. Zo kan je een aantal staten definieren van wat iets kan zijn en deze aanpassen met een automatisch gegenereerde methode. Je kan hier ook listeners mee laten genereren die iets kunnen doen wanneer de staat van jouw tekst/object veranderd.

  10. Google Maps - V3 utility library

    Met deze library kan je zelf je eigen applicatie maken aan de hand van Google's Google Maps service. De library geeft je de mogelijkheid deze widget te maken en er ook mee te werken. Heel handig als je iets wilt doen met geolocatie.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  11. Typescript

    TypeScript is niet echt een library, maar wel iets wat verder gaat dan standaard Javascript. TypeScript is een taal gebaseerd op Javascript, maar veranderd het in een stricte taal. - De syntax is dus precies hetzelfde, met de toevoeging dat je met types werkt in plaats van abstracte variabelen. Je kan de code die je schrijft omzetten naar gewone Javascript door gebruik te maken van een compiler. Hier bestaan tools voor in de meeste editors in de vorm van extensies. Heel interessant om iets mee te doen.

  12. Awesome-javascript

    Een collectie van javascript libraries. Leuk om mee te spelen, maar voor elke publicatie te groot omdat er zoveel onderdelen in zitten die je in het algemeen niet nodig hebt. Wel interessant om mee te spelen of doorheen te kijken als je benieuwd bent naar wat voor libraries er allemaal zijn.

Verklaring

Het animeren van elementen over een vectorpad is niet makkelijk te doen met standaard html/css/javascript, daarom heb ik de anime.js library gebruikt om dit voor elkaar te krijgen.

Het animeren van items over een vector pad is een mooie manier om artistieke elementen te verwerken op een webpagina, Dit is dan ook een simpele proof of concept van hoe dat eruit zou kunnen zien. Concepten die je hier ook op zou kunnen uitwerken is iets gelijksoortigs zoals knoppen die een pad volgen en ‘rondvliegen’ over een pagina. Het is zeker niet geschikt voor elke uitwerking, maar het is wel een mogelijkheid die je hebt met deze methode.

Uitleg code

Als eerst declareer ik een variabele genaamd ‘path’, ik geef deze een waarde die gegenereerd wordt met anime.js aan de hand van het pad element van mijn vector bestand. Deze gebruik ik vervolgens in de animatie methodes van anime.js om mijn ‘targets’ te laten animeren op het juiste pad. Deze targets hebben zelf ook nog een h1 element erin die ik ook willekeurig van grootte heb laten veranderen, ik heb hierbij keyframes moeten gebruiken zodat de grootte van de h1 elementen niet overspringt naar een andere waarde op het eind van de animatie. De animatie zelf duurt 10 seconden.

Ik heb naast de animaties die ik heb gemaakt ook nog wat ruimte gecreëerd tussen de letters door een setTimeout te benutten. Dit heb ik gedaan omdat de ingebouwde delay en offset attributen van anime.js bij een loop altijd wachten tot alle elementen klaar zijn met hun animatie loop, wat ervoor zorgt dat de letters na hun ronde over het pad “wachten” op de rest van de elementen, dit was niet gewenst maar ook niet op te lossen met anime.js dus heb ik een setTimeout gebruikt om de animaties later te laten starten en een offset te maken.

Three.js

Met three.js heb ik een draaiende kubus gemaakt die van kleur verandert.

Verklaring

De mogelijkheid om 3D graphics te kunnen renderen in je browser is een fantastische feature die veel mogelijkheden biedt, of het nou is voor een portfolio van een 3D designer, of een webshop die zijn producten wilt laten zien op een moderne wijze, Three.js is een handige library die je deze functionaliteiten biedt binnen Javascript zonder de benodigdheid van een IDE zoals Unity terwijl het wel alle functies biedt die je binnen zo’n IDE hebt..

Het poc dat ik heb gemaakt met het Three.js library is een test om te zien hoe ik elementen binnen de 3D omgeving kan manipuleren, in dit geval heb ik een kubus die draait een regenboog effect gegeven, en de camera die in de scene staat dichterbij de kubus gezet om hier echt de focus op te leggen.

Uitleg code

Ik heb een 3D scene gemaakt met behulp van de documentatie van Three.js, ik heb hier vervolgens aanpassingen op gedaan om ervoor te zorgen dat het past binnen mijn eigen omgeving. De achtergrond is zo bijvoorbeeld doorzichtig, het render venster heeft een ratio van 1:1 en wordt geplaatst op de juiste plek binnen mijn project. Vervolgens ben ik gaan kijken hoe ik een animatie kon toevoegen op de draaiende kubus, ik heb deze een kleuren animatie gegeven, hiervoor moest ik eerst uitzoeken hoe het materialen systeem van Three.js in elkaar zat.

Code

var threeJSCubeHuePos = 0;
function setThreeJSCubeHuePos(newValue) {
    threeJSCubeHuePos = newValue%360; return threeJSCubeHuePos;
}

var scene3D = new THREE.Scene();
var containerElement = document.getElementById("ThreeJSExample");
var camera = new THREE.PerspectiveCamera( 75, 320/320, 0.1, 1000 );

var renderer = new THREE.WebGLRenderer({alpha: true});
renderer.setSize( 320, 320 );
containerElement.appendChild( renderer.domElement );

var geometry = new THREE.BoxGeometry();
var material = new THREE.MeshBasicMaterial( { color: 0x00ff00 } );
var cube = new THREE.Mesh( geometry, material );
scene3D.add( cube );

camera.position.z = 1.5;

var animate = function () {
    requestAnimationFrame( animate );

    cube.rotation.x += 0.01;
    cube.rotation.y += 0.01;
    cube.material.color = new THREE.Color("hsl("+String(setThreeJSCubeHuePos(threeJSCubeHuePos+1))+", 100%, 50%)");

    renderer.render( scene3D, camera );
};

In week 3 ben ik bezig geweest met het uitwerken van library implementatie op mijn portfolio. Dit kwam in mijn geval voor de opdracht van week 2 omdat ik het nodig had voor het bouwen van dit portfolio waarop ik dus ook deze opdracht op kwijt moest.

Ik heb Anime.js gebruikt om een animatie probleem op te lossen dat niet op te lossen viel met standaard html en css, namelijk de hoogte van een container in- en uitklappen.

Product

De producten die ik heb gemaakt die gebruik maken van het in- en uitklappen zijn de artikels en de spoiler menu’s.

Het probleem

De transitie van vlakken in de breedte met percentages werkt uitmuntend, maar als ik dit deed in de hoogte dan zou de transitie ‘verspringen’. Dit gebeurde ook met anime.js, maar omdat anime.js per frame de grootte aanpast kon ik door het eerst uitrekenen van de hoogte van de content in pixels ook de animatie in pixels doen in plaats van alleen met percentages. Uiteindelijk kon ik door de meegeleverde ‘complete’ events binnen anime.js ook de hoogte van de vlakken na de animatie op 100% zetten zodat als het venster naderhand nog groter zou worden, deze alsnog mee zou bewegen.

Uitleg code

Ik heb een class gemaakt voor beide de artikel handler en de spoiler handler om alle methodes en variabelen bij elkaar te houden, dit zorgt ervoor dat er zo min mogelijk conflicten zullen ontstaan. Het belangrijkste onderdeel van beide classes zit ‘m in de calculateArticleSize methode binnen de ArticleHandler waarmee ik de pixelhoogte bereken van alle content binnen een element. Deze methode heb ik uiteindelijk ook gebruikt bij de spoilerHandler, ik moet ‘m dus nog uit de ArticleHandler halen en een andere benaming geven. Met deze methode kan ik de animaties van de spoiler en artikel elementen maken met de anime methode. Bij de spoilerHandler is dit het meest fast-forward. Ik speel de animatie af en nadat deze afgerond is zet ik met de complete event de hoogte van het spoiler element op 100%, hetzelfde gebeurt bij de articleHandler met een tussenstap. Dit is omdat de artikel elementen voordat ze in de hoogte uitklappen ook eerst nog in de breedte moeten uitklappen. Naast het vastzetten van de hoogte op 100% wordt er ook een class toegevoegd aan het element nadat deze is uitgeklapt (of weggehaald wanneer deze is ingeklapt), hiermee wordt niet alleen gekeken of het element actief is, maar ook welke styling er via css betrekking heeft tot dat element naast de hoogte.

class ArticleHander {
    articles;
    constructor() {}
    
    //
    getDocumentArticles() {
        this.articles = document.getElementsByTagName("article");
        return this.articles;
    }
    
    getDocumentActiveArticles() {
        var returnArray = new Array();
        Array.from(this.articles).forEach((article) => {
            if (article.classList.contains("active")) { returnArray.push(article); }
        })
        return returnArray;
    }
    
    assignActivation() {
        Array.from(this.articles).forEach(article => {
            article.onclick = () => {
                this.animeAnimateArticle(article, true);
                document.onclick = (event) => { // NEEDS REVISION, OVERRIDES ANY OTHER EVENTS.
                    if (!article.contains(event.target)) { this.animeAnimateArticle(article, false); }
                };
            }
        });
    }
    
    // ANIME.JS
    animeAnimateArticle(article, activate) {
        if (activate) {
            if (article.classList.contains("active")) { return; }
            console.log("Activating " + article);
            Array.from(this.getDocumentActiveArticles()).forEach((activeArticle) => {
                this.animeAnimateArticle(activeArticle, false);
            })
            article.classList.add("active");
            anime({
                targets: article,
                width: ["calc(50% - 20px)","100%"],
                easing: 'easeInOutQuad',
                duration: 600,
                complete: () => {
                    anime({
                        targets: article,
                        height: ["320px", this.constructor.calculateArticleSize(article)],
                        easing: 'easeInOutQuad',
                        complete: () => {
                            article.style.height = "100%";
                        }
                    });
                }
            });
        } else {
            if (!article.classList.contains("active")) { return; }
            console.log("Deactivating " + article);
            anime({
                targets: "article.active",
                height: [this.constructor.calculateArticleSize(article), "320px"],
                easing: 'easeInOutQuad',
                complete: () => {
                    article.classList.remove("active");
                    anime({
                        targets: article,
                        width: ["100%","calc(50% - 20px)"],
                        easing: 'easeInOutQuad',
                        duration: 600,
                        complete: () => {
                        }
                    });
                }
            });
        }
    }
    
    // UTIL
    static calculateArticleSize(article) {
        let articleSize = 0;
        articleSize += parseInt(window.getComputedStyle(article).paddingBottom.substr(0,window.getComputedStyle(article).paddingBottom.length-2));
        articleSize += parseInt(window.getComputedStyle(article).borderBottomWidth.substr(0,window.getComputedStyle(article).borderBottomWidth.length-2));
        Array.from(article.children).forEach((element) => {
            articleSize += parseInt(window.getComputedStyle(element).height.substr(0,window.getComputedStyle(element).height.length-2));
            articleSize += parseInt(window.getComputedStyle(element).marginBottom.substr(0,window.getComputedStyle(element).marginBottom.length-2));
        });
        return articleSize.toString() + "px";
    }
    
}
    
const articleHandler = new ArticleHander();

    articleHandler.getDocumentArticles();
    articleHandler.assignActivation();

class SpoilerHandler {
    spoilers;
    constructor() {}
    
    getSpoilers() {
        this.spoilers = document.getElementsByClassName("spoiler");
        return this.spoilers;
    }
    
    static toggleSpoiler(element) {
    
        var contentHeight = ArticleHander.calculateArticleSize(element.children[0]);
    
        if (!element.classList.contains("active")) {
            // Not active
            anime({
                targets: element,
                height: ["0", contentHeight],
                easing: 'easeInOutQuad',
                duration: 500,
                complete: () => {
                    element.classList.add("active");
                    element.style.height = "100%";
                }
            });
        } else {
            // Active
            anime({
                targets: element,
                height: [contentHeight, "0"],
                easing: 'easeInOutQuad',
                duration: 500,
                complete: () => {
                    element.classList.remove("active");
                }
            });
        }
    }
}
    
const spoilerHandler = new SpoilerHandler();
    
window.addEventListener('load', () => {
    
    Array.from(spoilerHandler.getSpoilers()).forEach( (spoiler) => {
    
        spoiler.addEventListener('click', (event) => {
            if(event.target == spoiler) {
                SpoilerHandler.toggleSpoiler(spoiler);
            }
        });
    
    });

De presentatie voor Glamorous Goat was een optioneel onderdeel voor ons project dat we hebben gedaan om onduidelijkheden over het project tijdig te verduidelijken.

Proces

We vonden de opdracht nog wat onduidelijk dus hebben we als team besloten om onderzoek te doen naar een aantal mogelijke concepten. Hieruit hadden we drie concepten gehaald die we in groepen van twee wouden gaan presenteren aan onze stakeholder; Glamorous Goat.

Concept

Ons concept was om een grijswatersysteem te ontwikkelen voor huishoudens waarmee we restwater kunnen hergebruiken, beide binnen het eigen huis en binnen een gemeente.

Voorbereiding

Na het ontwikkelen van concepten hebben we contact opgenomen met Glamorous Goat om een afspraak te maken voor onze pitches. Na een bevestiging te hebben gekregen zijn we begonnen met het uitwerken van onze pitches hebben we een middag geoefend, daarna hebben we onze pitches aan elkaar gepresenteerd en feedback op elkaar gegeven. Op de dag van de pitch hebben we dit nog een keer gedaan terwijl we de feedback meegenomen hadden ter voorbereiding.

De pitches

De pitches zelf verliepen soepel. We hadden kleine problemen met de hardware waarmee we de pitches opgenomen hadden ondanks we deze van tevoren hadden gechecked, maar uiteindelijk verliep het toch allemaal soepel. We hebben ook al onze feedback gedocumenteerd.

Conclusie

Het is ons gelukt om een antwoord te krijgen op onze vraag; Glamorous Goat wou een applicatie design voor een water meetsysteem. Geen ander concept ondanks ze onze concepten wel interessant vonden.

Leeruitkomsten

  • Communication

    We hebben met onze stakeholders contact genomen voor duidelijkheid over onze opdracht, dit aan de hand van een presentatie die we hebben gedocumenteerd met video en audio. Daarnaast hebben we de feedback gedocumenteerd.

Reflectie

Wat ik geleerd heb van dit project is dat je vaak dingen over het hoofd ziet ondanks je denkt dat je wel goed zit. Meerdere iteraties en het herhaaldelijk vragen van feedback heeft geleid tot een zeer sterk product, in dit geval een pitch.

Portfolio - Styling

Het moodboard geeft het algemene idee en thema aan van het design dat ik wil uitwerken in mijn eigen portfolio site. Deze voorbeelden zullen dus meer thematisch gericht zijn dan de voorbeeld portfolio’s waarbij de focus vooral lag op layout van portfolio’s. Het doel van dit moodboard is om vooral te kijken naar kleurgebruik en designs (details dat het geheel maken).

Proces

Ik ben begonnen met het bedenken wat ik nodig had voor mijn design en website thema. - Aan de hand van de kennis die ik had van mijn vorige opleiding wist ik dat een moodboard het perfecte medium is voor het definieren van een algemeen thema. Dus dat is waar ik ben begonnen!
Bij het maken van het moodboard had ik een aantal termen nodig die ik wou dat mijn portfolio uitstralen; ‘clean’, ‘modern’ en ‘whitespace’. Dit zijn allemaal heel generieke termen die opzich niet echt veel meerwaarde hebben, maar wel in de context van het ontwerpen van een moodboard. Ik heb hiervoor Google images gebruikt om wat inspirerende afbeeldingen te vinden en heb deze samengevoegd met behulp van Paint.net. Het moodboard heeft voor mijn de stijl van mijn portfolio design zeker een heel stuk verstevigt. Na het ontwerpen van mijn moodboard ben ik begonnen met het vaststellen van een kleurenthema. In het kleurenschema wou ik per se oranje verwerken. Dit is een persoonlijk aspect omdat oranje mijn favoriete kleur is, het straalt namelijk energie uit zonder aggressief over te komen, het is daarnaast ook een beetje speels. Ik heb deze kleur samen verwerkt naar één kleurenthema met een lichte variatie van oranje, een heel donker bruine en twee wit tinten. Ik heb hiervoor Adobe Color gebruikt. Ik heb het kleurenthema toegepast op een van de voorbeeld websites die ik in een vorige fase heb gevonden (Matt Farley’s portfolio site) en kwam al snel een probleem tegen; het was te druk. Na wat feedback te hebben gevraagd aan een medestudent en wat eigen brainstormen in combinatie met het moodboard dat ik heb gemaakt, heb ik een tweede kleurenthema gemaakt, het kleurenthema vervangt de lichtere tint oranje met een rood oranje, deze kleur gebruik ik vervolgens voor gradients. Deze design keuze maakt het geheel niet alleen rustiger, maar ook strakker en modern. Ik heb dit kleurenthema Red Fox genoemd en heb dit ook ontworpen in Adobe Color. Als je meer wilt lezen over dit proces, bekijk dan het officiele document!
